# 上海青少年AI教育日报网页站 Design

**日期：** 2026-06-04  
**状态：** 已确认设计，待实现计划  
**目标：** 将当前“Markdown 日报 + 微信发送”模式，改造成“Markdown 日报 + 自动生成静态网页日报站”模式，首页展示最新摘要、趋势/判断与历史归档，单日报提供完整详情页阅读体验。

---

## 1. 背景与问题

当前自动化已经能稳定产出 Markdown 日报，并按日期归档到 `reports/`。但微信发送链路反复受到桥接 DNS / 外网可达性影响，导致最终交付不稳定。与其继续依赖外部发送通道，不如把日报直接沉淀为一个长期可访问、可归档、可回看的静态网页站点。

这次改造不是换调研逻辑，而是换“最终展示层”：

- 保留现有 Markdown 日报作为原始档案；
- 取消微信发送作为每日交付目标；
- 新增自动网页生成步骤；
- 让首页负责“快速判断与导航”；
- 让详情页负责“完整阅读与证据追溯”。

---

## 2. 设计目标

### 2.1 业务目标

新站点需要持续帮助樱桃图书馆 / 登顶升学回答这些问题：

- 上海幼儿英语市场是否拥挤；
- 主要机构如何转向综合素养、AI、财商、项目制和双语表达；
- `AI + 财商 + 英语表达` 是否已经出现成熟案例；
- 我方体验日、五日营、长期班、高客单家庭方案可以如何差异化。

### 2.2 产品目标

站点必须做到：

- 每天自动产出一个独立日报详情页；
- 首页能快速看当天最重要的摘要；
- 首页有明显的“趋势/判断”区块，横向汇总最近几天核心结论；
- 首页能按时间浏览历史归档；
- 页面视觉要“好看”，但不是资讯门户或后台面板；
- 风格偏苹果官网式克制排版，同时保留暖灰玻璃感和研究刊物温度。

### 2.3 非目标

本次不做：

- CMS 后台；
- 数据库；
- 用户系统；
- 搜索；
- 评论；
- 可视化图表系统；
- 复杂交互筛选。

本次优先级是“自动稳定生成 + 页面高级感 + 长期归档可用”。

---

## 3. 站点信息架构

### 3.1 首页

首页文件路径：

- `index.html`

首页只做三件事：

1. 展示最新日报摘要；
2. 横向展示最近几天的趋势/判断；
3. 展示历史归档列表。

首页不直接展示完整正文，避免首屏过重，也避免网站退化成“把 Markdown 整段贴到首页”的弱实现。

### 3.2 每日报详情页

详情页路径：

- `daily/YYYY-MM-DD.html`

每个详情页对应一篇日报，完整呈现当天内容，重点优化阅读体验：

- 清晰标题与日期；
- 分章节阅读；
- 表格良好排版；
- 来源链接可点击；
- 页面顶部或侧边有章节导航；
- 有返回首页和前后期切换入口。

### 3.3 原始档案

继续保留现有 Markdown 档案：

- `reports/shanghai-youth-ai-edu-daily-YYYY-MM-DD.md`
- `shanghai-youth-ai-edu-daily.md`

网页只是展示层，不覆盖原始档案层。

---

## 4. 内容模型

为了让静态页面生成更稳定，站点需要从日报 Markdown 中提取少量结构化字段。

### 4.1 首页所需字段

每篇日报至少提取：

- `date`
- `title`
- `summaryBullets`
- `trendBullets`
- `archiveHeadline`
- `archiveSummary`
- `detailPath`

其中：

- `summaryBullets` 来自“今日结论”的前 3 条左右；
- `trendBullets` 来自“今日结论”中的高价值横向判断，供首页趋势区块使用；
- `archiveHeadline` 是适合放进归档列表的一句话标题；
- `archiveSummary` 是归档列表的一句补充说明。

### 4.2 详情页所需字段

详情页需要保留日报完整结构，包括：

- 今日结论
- 英语机构转型观察
- 机构与课程表
- AI+财商+英语表达专项观察
- 高频卖点/内容趋势
- 新增线索
- 对樱桃图书馆/登顶升学的启发
- 今日可测试动作
- X/Twitter英文趋势观察
- 渠道覆盖与失败说明

### 4.3 趋势/判断区块规则

首页趋势区块不是复制最近几篇的全部“今日结论”，而是做有限汇总。

第一版规则：

- 从最近 `3-5` 篇日报中提取高价值判断；
- 最终首页展示 `3` 张趋势卡；
- 趋势卡的语义优先聚焦：
  - 市场拥挤度
  - 产品白地
  - 竞争演化 / 产品形态变化

如果自动提取结果不完美，也允许第一版采用相对简单的规则，只要输出稳定、可读、不过度失真。

---

## 5. 视觉系统

### 5.1 总体方向

视觉方向已经确认：

- 参考苹果官网式排版；
- 不是深色科技风；
- 不是高噪声仪表盘；
- 不是花哨营销页；
- 是“克制、留白、轻层次、偏暖灰玻璃感”的研究刊物站。

### 5.2 气质关键词

- 克制
- 高级
- 安静
- 有判断力
- 适合长时间阅读
- 适合长期归档

### 5.3 具体视觉约束

- 背景不能过白，要有明显但克制的暖灰层次；
- 标题采用大字号、大留白、强层级；
- 卡片数量要少，避免首页碎片化；
- 阴影和玻璃感只能轻用，不能做成炫技；
- 强调文字排版和阅读节奏，而不是图表堆砌；
- 链接、按钮和重点交互可使用克制高亮色，但整体不应偏艳；
- 首页更像“研究刊物首页”，不是 SaaS dashboard。

### 5.4 组件风格

首页至少包含以下视觉模块：

- Hero / 站点头部
- 最新日报摘要区
- 趋势/判断卡片区
- 历史归档列表区

详情页至少包含：

- 标题头部
- 正文内容区
- 章节导航
- 来源链接样式
- 表格样式
- 页底导航

---

## 6. 自动生成流程

### 6.1 每日自动化流程的新终点

当前自动化流程大致是：

1. 读取缓存；
2. 调研；
3. 生成 Markdown 日报；
4. 同步最新 Markdown；
5. 尝试微信发送。

改造后应变为：

1. 读取缓存；
2. 调研；
3. 生成 Markdown 日报；
4. 同步最新 Markdown；
5. 生成站点数据；
6. 生成 / 更新网页站点；
7. 做基础校验；
8. 完成。

微信发送步骤从主流程中移除。

### 6.2 站点产物

站点构建后应至少生成：

- `index.html`
- `daily/YYYY-MM-DD.html`
- `assets/site.css`
- 可选 `assets/site.js`
- 可选 `site-data/reports.json`

如果实现上需要，还可以生成：

- `assets/` 下的图标或字体配置；
- `daily/index.html` 作为详情页目录跳转页。

### 6.3 增量更新规则

每次自动化运行时：

- 当天详情页必须重建；
- 首页必须重建；
- 历史归档列表必须重建；
- 最近几天趋势区块必须基于最新历史集合重算。

不要求复杂增量构建，因为页面规模很小，完全重建更简单、更稳。

---

## 7. 页面行为规则

### 7.1 首页行为

首页必须支持：

- 点击“最新日报”进入当天详情页；
- 点击历史条目进入对应详情页；
- 趋势区块纯展示，不强制要求可点击。

### 7.2 详情页行为

详情页必须支持：

- 返回首页；
- 阅读完整正文；
- 点击来源链接跳转原始来源；
- 如果可行，支持“上一篇 / 下一篇”切换。

### 7.3 响应式要求

必须兼顾：

- 桌面大屏阅读；
- 笔记本正常浏览；
- 手机可读，但不要求做复杂移动交互。

移动端的重点是“可读”和“不断版”，不是完全复刻桌面布局。

---

## 8. 文件与职责拆分

建议拆分如下：

- `index.html`
  - 站点首页产物
- `daily/`
  - 每日报详情页产物
- `assets/site.css`
  - 全站样式
- `assets/site.js`
  - 少量必要交互
- `site-data/reports.json`
  - 首页与归档的结构化输入
- `scripts/` 下新增生成脚本
  - 负责读取 Markdown、提取结构、输出 HTML

如果脚本规模增长，允许拆成：

- Markdown 解析模块
- 首页渲染模块
- 详情页渲染模块
- 数据汇总模块

但第一版不应过度工程化。

---

## 9. 兼容旧流程的修改点

### 9.1 自动化提示词

`automation-prompt.md` 需要更新：

- 删除微信发送要求；
- 增加网页输出要求；
- 明确日报完成后必须生成网页站点；
- 明确需要更新首页、详情页与历史归档。

### 9.2 自动化记忆

后续每次自动化运行写入 memory 时，状态描述应改为：

- 日报 Markdown 是否已落盘；
- 网页首页是否已更新；
- 当天详情页是否已生成；
- 历史归档是否已刷新；
- 是否完成站点校验。

不再把“微信是否发送成功”作为完成条件。

---

## 10. 验证标准

实现完成后，至少要能验证：

1. 最新日报 Markdown 生成后，网页首页同步更新；
2. 当天详情页存在且可打开；
3. 首页能看到当天摘要；
4. 首页趋势区块能显示最近几天判断；
5. 首页历史归档列表包含已有历史日报；
6. 详情页完整展示 Markdown 核心内容；
7. 链接与表格样式可读；
8. 桌面与手机宽度下都不崩；
9. 不再依赖微信发送链路。

---

## 11. 风险与边界

### 11.1 内容解析风险

Markdown 日报是自然语言内容，结构虽然稳定，但并非严格机器模板。第一版在解析“今日结论”“趋势/判断”等字段时，应该依赖现有稳定标题格式，而不是做脆弱的自由文本理解。

### 11.2 视觉过度风险

如果实现时过度追求视觉花样，页面会偏离“研究刊物”定位，变成营销页。这次设计明确要求优先保证文字排版与阅读节奏。

### 11.3 自动化耦合风险

如果把网页生成逻辑直接硬编码进调研脚本里太深，后面维护会麻烦。应尽量让“日报生成”和“网页渲染”保持清晰分层。

---

## 12. 推荐实施策略

推荐分两段做：

### 第一段

先打通最小链路：

- 首页
- 当天详情页
- 历史归档
- 暖灰苹果风视觉

### 第二段

再补强：

- 趋势/判断自动提炼质量
- 上一篇 / 下一篇导航
- 更细致的 Markdown 表格与链接样式

这样可以先完成稳定交付，再逐步打磨。

---

## 13. 最终结论

本次改造采用：

- `静态刊物站 + 自动生成` 方案；
- 首页展示摘要、趋势/判断、历史归档；
- 详情页展示完整单日报；
- 视觉采用“偏苹果官网的克制排版 + 暖灰玻璃感背景”；
- 保留 Markdown 作为原始档案；
- 移除微信发送作为交付终点。

这是当前自动化目标下最稳、最轻、最可持续的方案，同时也为后续升级到更结构化的前端展示保留了空间。
